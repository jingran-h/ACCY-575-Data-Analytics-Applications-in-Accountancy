# Module 3 — Pull Requests and Code Review

> **Hands-on:** [walkthrough.md](walkthrough.md) — opening your first pull request and pair-reviewing a classmate's.

## Motivation

Last module you merged your own branches into main, no questions asked. Fine for a personal project. The minute you're on a team that doesn't fly — someone else has to read your change before it ships.

You already know this dynamic from accounting. You don't post an adjusting entry without a senior reviewing it. Code review is the same idea, just for a different kind of work product. Mechanically it's not hard. The actual skill this module is learning to write a PR description that doesn't waste a reviewer's time, and learning to leave review comments that are useful instead of pedantic.

This module is less new tools, more *practice*. The mechanics of opening a PR are simple. The hard part is writing useful PR descriptions, leaving useful review comments, and reading code that someone else wrote.

## Goals

- Open a pull request with a description that lets a reviewer understand it without reading every diff.
- Leave code-review comments that are specific, kind, and actionable. (You'll review a classmate's PR this module — the same skill you'll use in Part 2 to evaluate AI-generated code.)
